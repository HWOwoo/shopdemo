# 프로젝트 작업 규칙

## 문서 관리

- **해야 할 작업 목록** → `TODO.md` 에 기록
- **기능 명세 / 작업 내용** → `REQUIREMENTS.md` 에 기록

### TODO.md 작성 규칙
- 구현할 기능, 버그 수정, 개선 사항을 항목으로 나열
- 완료된 항목은 체크 표시 또는 삭제

### REQUIREMENTS.md 작성 규칙
- 구현된 기능의 상세 명세, 흐름, 엔티티 구조 등을 기록
- 새 기능 구현 시 해당 섹션을 추가하거나 기존 섹션을 업데이트
- 미완성 항목은 상태를 명시 (❌ 미구현 / ✅ 완료)
